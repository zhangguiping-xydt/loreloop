# Support

Use GitHub Discussions for setup and workflow questions, and the issue
templates for reproducible bugs or focused feature requests. Early-alpha
support is best effort; there is no response-time or production SLA.

Before asking for help, run `loreloop doctor`, review
[`docs/troubleshooting.md`](docs/troubleshooting.md), and include sanitized
versions, platform details, and reproduction steps. Never post credentials,
private repository contents, or unredacted `.loreloop/evidence/` artifacts.

Report security vulnerabilities through GitHub Security Advisories as
described in [`SECURITY.md`](SECURITY.md).
