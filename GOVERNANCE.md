# Governance

LoreLoop is currently maintainer-led. Maintainers set release scope, review
changes, protect the trust model, and may appoint additional maintainers after
sustained high-quality contributions.

Routine changes require one maintainer approval and passing required checks.
Changes to evidence formats, trust projection, signing boundaries, browser
write controls, or backward compatibility require a written design note and
review from two maintainers when two are available. Security fixes may be
prepared privately and released before public disclosure.

Decisions prefer documented technical reasoning and rough consensus. If
consensus is not reached, the release maintainer makes the decision and records
the rationale. Governance changes are proposed through pull requests.
